# Reflection

## 1. HTTP Request Path

When I open my site in a browser, the request starts with the browser looking up `omaimasharhan.github.io` to find the server’s IP. It then connects to GitHub’s server, requests the files like `index.html` and CSS, and the browser receives them and displays the website.

---

## 2. Docker vs GitHub Pages

Docker provides a full environment where I can run programs with all the dependencies and system settings I need. GitHub Pages, on the other hand, only serves static files like HTML, CSS, and JavaScript, so it cannot run backend code or applications.
